## 這個懶人包會幫你做什麼？

這是一份「Excel 月報維護包」。適合像製造部、班級營運或行政報表這類每月都會新增資料、但 Excel 裡已經有大量公式與分頁邏輯的工作簿。

它會讓 Codex 先把 Excel 當成正式系統來盤點，而不是只把它當成可以任意改寫的檔案。

主要用途：

- 整理每月來源檔，例如 `4月実績.xls`
- 排除 `合計` 列，將重複品目加總
- 把每月台數寫入主檔指定月份欄
- 保留既有 ST、能率、工數公式
- 整合可閱讀總覽
- 把底層計算分頁隱藏但不刪除
- 新增 `月別設定` 分頁，集中維護每月可變條件
- 避免 Excel 開啟時出現「部分內容有問題，是否修復」警告

```text
  2026製品部人員編制&能率(最新).xlsx
```

## 建議工作流程

### 1. 先盤點，不要直接改

先列出：

- 來源檔分頁
- 主檔分頁
- 每個分頁的使用範圍
- 公式數量
- 公式引用到哪些分頁
- 哪些分頁是輸入、哪些是計算、哪些是閱讀用總覽

Codex 回報時要用這種分類：

| 類型 | 分頁例 |
|---|---|
| 月台數輸入 | `台數&ST` |
| ST 主資料 | `年計用ST2026` |
| 每月條件輸入 | `月別設定` |
| 底層計算 | `1係`, `2係`, `3係`, `加工`, `塗裝`, `製品課合計`, `製品部合計`, `加工塗裝` |
| 閱讀總覽 | `整合總覽` |

---

### 2. 整理來源檔

來源檔常見不是單一表，而是橫向多組欄位。

實測案例：

```text
A:C = 品目コード / 品名 / 加工入庫数量
E:G = 品目コード / 品名 / 加工入庫数量
I:K = 品目コード / 品名 / 加工入庫数量
M:O = 品目コード / 品名 / 加工入庫数量
```

整理規則：

1. 掃描整張表，找出所有 `品目コード / 品名 / 加工入庫数量` 欄位組。
2. 不要只抓第一組或最後一組。
3. 排除 `合計` 與旁邊數量。
4. 相同 `品目コード + 品名` 要加總。
5. 依 `コード` 升冪排序。
6. 輸出新版 `.xlsx`，不要覆蓋舊 `.xls`。

輸出分頁範例：

| コード | 品名 | 加工入庫数量 |
|---|---|---:|
| 884T8050501 | BH8000L | 96 |

---

### 3. 對應到主檔

以 `台數&ST` 為主資料表。

| 月份 | 欄位 |
|---|---|
| 4月 | E |
| 5月 | F |
| 6月 | G |
| 7月 | H |
| 8月 | I |
| 9月 | J |
| 10月 | K |
| 11月 | L |
| 12月 | M |
| 1月 | N |
| 2月 | O |
| 3月 | P |

更新規則：

1. 用來源 `品目コード` 對主檔 `台數&ST` 的 `機種コード:`。
2. 找到既有 code，就把該月台數寫入對應月份欄。
3. 找不到 code，就新增到 `台數&ST` 的 D 欄。
4. 新增 code 後，Q:X 的 ST 公式要延伸。
5. 若 `年計用ST2026` 沒有該 code，Q:X 可以顯示 `無此資料`，但要記錄清楚。

## 最重要的設計：不要刪底層計算分頁

實測後確認：

只靠 `整合總覽` 不能完全取代底層計算分頁。

原因：

- `1係 / 2係 / 3係 / 加工 / 塗裝` 有每月輸入條件與公式。
- `製品課合計 / 製品部合計 / 加工塗裝` 是中間彙總層。
- 如果直接刪除，總覽可能看起來還有值，但其實只是快照，未來切換月份或新增資料時邏輯會失真。

正確做法：

- 保留底層計算分頁。
- 設成 `Very Hidden`，一般 Excel 分頁列看不到。
- 新增可見的 `月別設定` 分頁，集中維護每月輸入條件。
- `整合總覽` 只做閱讀與月份切換。

## 建議可見分頁

正式維護版建議只讓使用者看到 5 個分頁：

| 分頁 | 用途 |
|---|---|
| `整合總覽` | 看結果、切換月份 |
| `月別設定` | 每月調整生產天數、人員、出勤、加班、扣除時數 |
| `標準工數` | 基準工數設定 |
| `台數&ST` | 每月台數輸入與 ST 對照 |
| `年計用ST2026` | ST 主資料 |

底層分頁設成 `Very Hidden`：

```text
製品部合計
製品課合計
1係
2係
3係
加工塗裝
加工
塗裝
```

## 月別設定分頁

`月別設定` 是這次實測後最重要的修正。

每月可調整欄位集中在這裡：

| 欄位 |
|---|
| 月份 |
| 單位 |
| 生產天數 |
| 工程數 |
| 直接人數 |
| 準間接人數 |
| 出勤率 |
| 欠勤時間 |
| 晚上加班天數 |
| 晚上加班人數 |
| 白天加班天數 |
| 白天加班人數 |
| 扣除時數 |

另外保留一列或區塊維護：

- `製品課其他人員`

底層 `1係 / 2係 / 3係 / 加工 / 塗裝` 的輸入欄應改成引用 `月別設定`，不要讓使用者必須打開底層分頁。

## 整合總覽分頁

建議分成四段。

### 1. KPI 區

- 月份選擇
- 年度台數合計
- ST 未對應數

### 2. 年度月別台數

橫向顯示：

```text
4月, 5月, 6月, 7月, 8月, 9月, 10月, 11月, 12月, 1月, 2月, 3月
```

### 3. 單月部門摘要

欄位建議：

| 項目 | 直接/資料 | 其他人員 | 實働工數 | 實績工數 | 標準工數 | 總能率 | 直接能率 | ロス率 |
|---|---:|---:|---:|---:|---:|---:|---:|---:|

### 4. 單月各單位摘要

欄位建議：

| 單位 | 直接人數 | 準間接人數 | 實働工數 | 實績工數 | 標準工數 | 總能率 | 直接能率 | ロス率 |
|---|---:|---:|---:|---:|---:|---:|---:|---:|

> 注意：不要漏掉「直接能率」。這次實測曾漏掉，使用者會看不出每個單位的直接效率。

## 重要坑位與修正

| 問題 | 原因 | 修正 |
|---|---|---|
| A:M 資料沒整理到 | 來源表橫向有多組欄位，只抓到最後一組 | 掃描所有 `品目コード / 品名 / 加工入庫数量` 欄位組 |
| Excel 開啟跳「部分內容有問題」 | 用非 Excel 原生方式直接儲存複雜 `.xlsx`，破壞部分內部結構或公式快取 | 複雜主檔優先用 Excel COM 開啟、計算、儲存；必要時用 Excel 修復另存 |
| Q:X 公式存在但看不到數值 | 公式快取值被清掉或 Excel 沒重新計算 | 用 Excel `CalculateFullRebuild()` 重新計算；不要把複雜公式隨意改成值 |
| 新增機種後 Q:X 空白 | 新增列放在公式範圍外，沒有延伸公式 | 新增列放在正確資料區，或把 Q:X 公式延伸到新增列 |
| 找不到 ST 的新增機種 | `年計用ST2026` 沒有該 code | 顯示 `無此資料` 並記錄，不要硬補錯誤 ST |
| 刪除底層計算分頁後，月份切換邏輯失真 | 總覽只剩快照值，不再依原公式重算 | 不刪底層分頁，改設 `Very Hidden` |
| 完全扁平化動態版看似可切月份但邏輯不對 | 它查的是搬入總覽的快照表，不是完整公式鏈 | 正式維護版改用 `可更新極簡版 + 月別設定` |
| 使用 XLOOKUP 後被轉成 `_xludf.XLOOKUP` | Excel 修復或版本差異導致公式被視為自訂函數 | 跨版本公式優先使用相容性較高的 `INDEX + MATCH`，或用 Excel 自身寫入公式並驗證 |
| 底層分頁 Very Hidden 後不能調整每月條件 | 每月可調整欄位仍藏在底層分頁 | 新增 `月別設定` 可見分頁，底層輸入欄引用它 |

## Codex 實作原則

### 對來源 `.xls`

可以用 Excel COM 或其他安全轉檔方式整理成 `.xlsx`。

### 對複雜主檔 `.xlsx`

不要一開始就用 `openpyxl` 或一般試算表程式庫直接大量改主檔。

建議順序：

1. 先建立副本。
2. 用 Excel 原生方式開啟副本。
3. 寫入資料。
4. `CalculateFullRebuild()`。
5. 儲存。
6. 再用 Excel 背景開啟確認不跳修復。
7. 檢查公式結果與來源加總一致。

### 驗證至少要做

- Excel 可正常開啟
- 來源台數合計 = 主檔月份欄合計
- `整合總覽` 選 4月時數值正確
- 切到 5月或其他月份時，畫面會跟著月份變
- 有 ST 對應的 code，Q:X 不應空白
- 無 ST 對應的 code，明確顯示 `無此資料`
- 可見分頁符合預期
- 底層分頁仍存在且為 `Very Hidden`

## 建議交付版本命名

| 版本 | 用途 |
|---|---|
| `_重建版.xlsx` | 從原檔修復並保留公式 |
| `_整合版.xlsx` | 增加整合總覽、隱藏底層分頁 |
| `_可更新極簡版.xlsx` | 可見分頁少，但保留完整底層邏輯 |
| `_可更新極簡版_月別設定.xlsx` | 正式推薦維護版 |
| `_完全扁平化整合版.xlsx` | 只適合快照留存，不適合每月更新 |

正式維護建議使用：

```text
可更新極簡版_月別設定.xlsx
```

## 每月使用方式

1. 把新月份來源檔交給 Codex，例如 `5月実績.xls`。
2. Codex 整理來源檔，排除合計、重複加總。
3. Codex 寫入 `台數&ST` 對應月份欄。
4. 若有新機種，新增到 `台數&ST`。
5. 在 `月別設定` 調整該月人員、出勤、加班、扣除時數。
6. 到 `整合總覽` 的 `月份選擇` 切到該月。
7. 驗證台數合計、能率與 ST 對應。

---

## 最後提醒

這類 Excel 工作簿本質上是「小型業務系統」，不是單純表格。

不要只追求刪分頁。真正安全的目標是：

- 使用者看得到的分頁少
- 每月輸入集中
- 底層公式保留
- 可驗證
- 可回復
- 不破壞 Excel 原生結構
