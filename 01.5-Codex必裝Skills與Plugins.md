## 初學者必裝清單

### 必裝外掛程式

| 外掛程式 | 用途 | 老師使用場景 |
|--------|------|--------------|
| GitHub | 讀 repo、查 issue / PR、協助同步程式碼 | 發布教學網頁、管理懶人包 |
| Browser Use | 開網頁、測 localhost、看畫面、截圖 | 測互動教材、檢查網頁畫面 |
| Documents | 建立 / 修改 Word | 試卷、講義、研習文件 |
| Presentations | 建立 / 修改 PowerPoint | 研習簡報、教學投影片 |
| Spreadsheets | 建立 / 修改 Excel、CSV | 成績表、題庫、雙向細目表 |

### 推薦技能

| 技能 | 用途 | 老師使用場景 |
|-------|------|--------------|
| imagegen | 生成或修改圖片 | 封面圖、教材插圖、遊戲素材 |
| openai-docs | 查 OpenAI / Codex 官方文件 | 避免照過期教學設定 |
| skill-installer | 安裝其他 skills | 擴充 Codex 能力 |
| skill-creator | 建立自己的 skills | 把重複教學流程變成 SOP |

### 需要再裝 / 進階再裝

| 項目 | 建議 |
|------|------|
| Gmail | 需要整理信件、草擬回信再連 |
| Google Calendar | 需要行程摘要、會議準備再連 |
| Obsidian / MCPVault | 第二大腦主題，建議留到後續集數 |
| 其他 MCP | 不知道用途就先不要裝，本集不講 |

## 步驟零：先做基礎工具快篩

外掛程式與技能可以讓 Codex 變強，但有些基礎工具是整個工作流的地基。請 Codex 先檢查：

```powershell
git --version
gh --version
node --version
uv --version
```

請整理成表格：

```md
| 工具 | 狀態 | 為什麼需要 |
|------|------|------------|
| Git | 已安裝 / 未安裝 | 專案版本控制、commit、push |
| GitHub CLI (`gh`) | 已安裝 / 未安裝 | GitHub 登入、建立 repo、Pages |
| Node.js | 已安裝 / 未安裝 | 後續很多 MCP / 工具會用到 |
| uv | 已安裝 / 未安裝 | Python 工具與部分 MCP 安裝 |
```

### 如果 Git 沒有安裝

Windows：

```powershell
winget install --id Git.Git --accept-source-agreements --accept-package-agreements
```

macOS：

```powershell
xcode-select --install
```

安裝後重新確認：

```powershell
git --version
```

> [!warning]
> 安裝 Git 後，如果 Codex 還是找不到 `git`，請關閉並重新開啟 Codex Desktop app，再檢查一次。

### 如果 GitHub CLI 沒有安裝

Windows：

```powershell
winget install --id GitHub.cli --accept-source-agreements --accept-package-agreements
```

安裝後重新確認：

```powershell
gh --version
```

### 如果 Node.js 沒有安裝

Windows：

```powershell
winget install --id OpenJS.NodeJS --accept-source-agreements --accept-package-agreements
```

安裝後重新確認：

```powershell
node --version
```

### 如果 uv 沒有安裝

Windows PowerShell：

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

安裝後重新確認：

```powershell
uv --version
```

> [!tip]
> 如果缺很多工具，建議直接回去執行 [[Codex 懶人包/00-環境建置]]。本篇只做快篩與補救，不取代完整環境建置。

## 步驟四：GitHub 外掛程式與本機 GitHub CLI 要分清楚

雖然本集主線只講外掛程式與技能，但 GitHub 會同時牽涉本機工具，所以要先埋一個觀念：

| 類型 | 用途 | 檢查方式 |
|------|------|----------|
| GitHub CLI (`gh`) | commit / push / 建 repo / GitHub Pages | `gh auth status` |
| GitHub 外掛程式 | 讓 Codex 對話中讀 repo / issue / PR | 請 Codex 列出 GitHub 帳號與 repo |

檢查 GitHub CLI：

```powershell
gh auth status
```

成功時應看到：

```text
Logged in to github.com account <你的帳號>
Git operations protocol: https
Token scopes: ... repo ...
```

檢查 GitHub 外掛程式是否連上帳號：

```text
幫我確認 GitHub 外掛程式是否已連接，並列出我可以存取的 GitHub 帳號與幾個 repo。
```

如果出現：

```text
accounts: []
installations: []
repositories: []
```

代表 Codex Desktop 的 GitHub 外掛程式還沒連上帳號。下一集用 [[Codex 懶人包/02-連接-GitHub]] 解決。

## 新手常見問題

| 問題 | 解法 |
|------|------|
| 技能跟外掛程式差在哪 | 技能是工作流程；外掛程式是能力包 |
| GitHub 外掛跟 GitHub CLI 差在哪 | 外掛讓 Codex 對話讀 GitHub；CLI 讓本機 commit / push |
| 為什麼本集又檢查 Git | 因為完全新手常跳過 #00，沒有 Git 後面 GitHub 工作流會直接卡住 |
| 一定要裝 Gmail / Calendar 嗎 | 不一定，需要整理信件或行程再裝 |
| MCP / Obsidian 要本集就裝嗎 | 不建議，留到第二大腦集數 |
| 我看不到某個外掛 | 可能尚未啟用 marketplace，或 Codex 版本不同 |
| 裝越多越好嗎 | 不是，先裝高頻、安全、用得到的 |

## 2026-05-07 Codex Desktop 實測補充：macOS 快篩坑

這次在 macOS + Codex Desktop 實測時，最容易卡住的不是 Codex 本身，而是基礎工具路徑與登入狀態。

| 狀況 | 原因 | 解法 |
|------|------|------|
| `gh` 找不到 | macOS 可能沒有 Homebrew，也未安裝 GitHub CLI | 可用 Homebrew 安裝，或下載 GitHub CLI 官方 release 後放到 `~/.local/bin/gh` |
| `uv` 找不到 | Codex 不一定內建 uv | 用官方 installer 安裝到使用者層，通常會建立 `~/.local/bin/env` |
| 已安裝 `uv` / `gh` 但 Codex 找不到 | Codex 的非互動 shell 不一定讀 `~/.zprofile` 或 `~/.zshrc` | 在 `~/.zshenv` 加上 `source "$HOME/.local/bin/env"`，讓 Codex shell 也吃到 PATH |
| `node` 有但 `npm` / `npx` 沒有 | Codex 內嵌 Node 或系統 Node 不一定附完整 npm 工具鏈 | 安裝官方 Node.js，確認 `node --version`、`npm --version`、`npx --version` 都可執行 |
| `gh auth status` 顯示失敗，但瀏覽器剛完成登入 | 沙盒或網路權限可能讓 GitHub CLI 驗證誤判 | 用可連網的終端機重跑 `gh auth status`；不要只看第一次沙盒錯誤就判定登入失敗 |
| GitHub 外掛已啟用，但 `git push` 仍失敗 | 外掛啟用與 GitHub CLI 授權是兩件事 | 外掛負責 Codex 連 GitHub；本機 commit / push 仍要 `gh auth login` 或有效 Git 憑證 |
