> - 建立

---

### 步驟二之二：設定 Firestore 安全規則（自動化）

請 Codex 在使用者的專案資料夾建立三個檔案：

**1. `firestore.rules`**：
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /wordcloud_words/{document} {
      allow read, write: if true;
    }
    match /{document=**} {
      allow read, write: if false;
    }
  }
}
```

**2. `firebase.json`**：
```json
{
  "firestore": {
    "rules": "firestore.rules"
  }
}
```

**3. `.firebaserc`**：
```json
{
  "projects": {
    "default": "[使用者的 Firebase 專案 ID]"
  }
}
```

部署規則：
```powershell
npx.cmd -y firebase-tools@latest deploy --only firestore:rules
```

成功會看到 `Deploy complete!`。

> 💡 之後新工具：編輯 `firestore.rules` 加 `match /votes/{document} { allow read, write: if true; }` → 對 Codex 說「部署 Firestore 規則」 → Codex 自動 `firebase deploy --only firestore:rules`。

## 階段二：連接 MCP

### 步驟三：把 Firebase 註冊為 Codex 的 MCP server

> ✅ 三種 Codex 共用 `~/.codex/config.toml`。

**方法 A：Codex Desktop GUI（推薦）**

1. Desktop → 設定 → **Integrations & MCP** → **Add server**
2. 填：
   - Name：`firebase`
   - Command：Windows 建議填 `npx.cmd`；macOS / Linux 填 `npx`
   - Args：`-y`、`firebase-tools@latest`、`mcp`
3. 儲存

**方法 B：手動編輯 `~/.codex/config.toml`**

```toml
[mcp_servers.firebase]
command = "npx.cmd"
args = ["-y", "firebase-tools@latest", "mcp"]
startup_timeout_sec = 60
tool_timeout_sec = 120
```

> Windows 建議用 `npx.cmd`，比較不會踩到 PowerShell `.ps1` 執行原則問題。macOS / Linux 可用 `command = "npx"`。

**方法 C：CLI**

```bash
codex mcp add firebase -- npx -y firebase-tools@latest mcp
```

> 2026-04-29 實測：本機 Codex Desktop 可用「方法 B」成功載入。若 `codex` CLI 在 Windows 顯示 `Access is denied`，不要卡在 CLI，改用 GUI 或手動改 `config.toml`。

### 步驟六：功能測試

1. 列出 Firebase 專案
2. 列出目前 Firestore 集合
3. 在 Firestore 建集合 `test_collection`，新增一筆 `message`「Firebase 連接測試成功」
4. 查 `test_collection` 確認能讀
5. 刪除測試文件
6. 再讀一次，確認文件不存在
7. ✅ 「連接測試成功！」

> 若你的 `firestore.rules` 只白名單開放特定集合，例如 `wordcloud_words`，測試時要改用已允許的集合，不要硬測 `test_collection`。否則會誤判成 Firebase 或 MCP 壞掉，其實只是 rules 擋住。

## 如果失敗

對 Codex 說：「Firebase 懶人包失敗，幫我檢查重新處理。」

完全重置：
- **Desktop**：設定 → Integrations & MCP → 刪 `firebase`
- **手動**：編輯 `~/.codex/config.toml` 刪 `[mcp_servers.firebase]` 段
- **CLI**：`codex mcp remove firebase`

重登 Firebase：
```powershell
npx.cmd -y firebase-tools@latest logout
npx.cmd -y firebase-tools@latest login
```

## 常見問題

| 問題 | 原因 | 解法 |
|------|------|------|
| `npx: command not found` | Node.js 或 npm/npx 沒有正確安裝到 PATH | 確認 Node.js 已裝，重啟終端機 |
| PowerShell 顯示 `npx.ps1 cannot be loaded because running scripts is disabled` | Windows 執行原則擋到 `npx.ps1` | 改用 `npx.cmd --version`、`npx.cmd -y firebase-tools@latest ...` |
| Codex 讀 Google Drive 專案資料夾出現 sandbox 權限錯誤 | Codex 需要使用者授權讀寫該路徑 | 同意授權後重跑；不要先判定檔案壞掉 |
| 查詢失敗 | 可能尚未登入 Firebase CLI，或 MCP session 尚未重啟 | 確認 `firebase login` 已成功，必要時重啟 Codex Desktop |
| `firestore_query_collection` 回 `read_time cannot be in the future` | MCP 查詢工具可能送出未來時間參數 | 先用 `firestore_get_document` 或 `firestore_list_documents` 驗證可讀取，必要時重啟 MCP 後再查 |
| 看不到 Firestore 資料 | Firestore 尚未啟用或專案 ID 不對 | 確認 Console 已啟用 Firestore，並用 `projects:list` 確認 `.firebaserc` 指到目前帳號可見專案 |
| 安全性規則過期 | 用了測試模式 | 改白名單規則（步驟二之二） |
| `Permission denied` | rules 沒允許這個集合 | 改測 rules 已允許的集合，或先更新並部署 `firestore.rules` |
| `Collection id is invalid because it contains /` | `collection_path` 或 `collectionId` 寫成含 `/` 的路徑 | 寫 `wordcloud_words`，不要寫 `wordcloud_words/` |
| `firebase login` 在對話中卡住 | 互動式登入無法在 Codex 對話內完成 | 開 cmd/PowerShell 手動跑 `npx.cmd -y firebase-tools@latest login` |
| Codex 重開前看不到 Firebase 工具 | 改完 `~/.codex/config.toml` 後目前 session 尚未載入 MCP server | 完整重開 Codex Desktop；只在同一輪對話重新搜尋通常不會載入新 MCP server |
| Windows 跑 `npx` 出問題 | PowerShell 或 repo 路徑可能讓 `npx` 解析到錯誤入口 | 把 MCP 設定的 `command` 改成 `npx.cmd`；必要時用完整路徑 `C:\Program Files\nodejs\npx.cmd` |
| `codex --version` 顯示 `Access is denied` | Windows CLI 權限問題，不一定影響 Desktop MCP | 不要卡在 CLI；改走 GUI 或手動編輯 `~/.codex/config.toml` |
| `firestore_add_document` 說 document 格式 invalid | 把 Firestore document 包成 JSON 字串 | 傳 document 物件格式，例如 `fields.message.stringValue` |
| CRUD 測試新增成功但讀不到或被拒絕 | 測試集合沒有被 `firestore.rules` 白名單允許 | 用 `wordcloud_words` 這類已允許集合測試，或先把測試集合加進 rules 並部署 |
| 測試後資料庫留下垃圾資料 | 只做新增/讀取，沒有清理測試文件 | 測完呼叫 `firestore_delete_document`，再用 `firestore_get_document` 確認回報 `not found` |
