## 這個懶人包會幫你做什麼？

這是一份「工作模式啟動包」。跑完之後，你就可以用 Codex 管理一個教學應用專案：

- 建好 Google Drive / OneDrive 裡的工作資料夾
- 建好 `AGENTS.md`，讓 Codex 每次都知道專案規則
- 建好 Obsidian 專案駕駛艙，記錄進度、下一步、踩坑
- 建好 GitHub repo，必要時啟用 GitHub Pages
- 套用 Codex 的三個全域流程：
  - `project-init-sync`：新專案初始化
  - `startup-sync`：開工接續
  - `shutdown-sync`：收工同步
- 讓專案準備好，接著回到影片教學，跟著做第一個工具

> 一句話：Claude 版靠 `/開工`、`/收工` skills；Codex 版改用內建的 `開工`、`收工`、`新專案初始化` 三個全域 skill 流程。

## GitHub repo 建立流程

若要用 GitHub Pages，建議公開 repo。

PowerShell 範例：

```powershell
Set-Location "G:\我的雲端硬碟\<專案名稱>"
git init
git config windows.appendAtomically false
git add AGENTS.md README.md .gitignore
git commit -m "初始化教學工具專案"
git branch -M main
gh repo create <repo名> --public --source=. --push
```

如果不需要 Pages，可以改成私有：

```powershell
gh repo create <repo名> --private --source=. --push
```

> Google Drive 內的 git repo 建議先設 `git config windows.appendAtomically false`，可避免 `cannot update the ref` 之類的同步衝突。

## 常見問題

| 問題 | 可能原因 | 解法 |
|---|---|---|
| `gh repo create` 失敗 | GitHub CLI 尚未登入或 token 權限不足 | 先跑 `gh auth status`，未登入就 `gh auth login` |
| GDrive 內 commit 報錯 | Google Drive 同步與 Git ref 寫入衝突 | 跑 `git config windows.appendAtomically false` 後重試 |
| 新 repo 跑 `git config windows.appendAtomically false` 出現 `fatal: not in a git directory` | 還沒初始化 Git repo | 先 `git init`，再跑 `git config windows.appendAtomically false` |
| 文件寫 main 但 Git 顯示 master | Git 預設分支名稱不同 | 第一次 commit 後執行 `git branch -M main` |
| Windows 上「開工」沒觸發 | `startup-sync` skill 不存在或 Codex 尚未重啟 | 確認 `C:\Users\<你>\.codex\skills\startup-sync\SKILL.md` 存在，必要時建立後重開 Codex |
| Windows 上「收工」沒觸發 | `shutdown-sync` skill 不存在或 Codex 尚未重啟 | 確認 `C:\Users\<你>\.codex\skills\shutdown-sync\SKILL.md` 存在，必要時建立後重開 Codex |
| Windows 上「新專案初始化」沒觸發 | `project-init-sync` skill 不存在或 Codex 尚未重啟 | 確認 `C:\Users\<你>\.codex\skills\project-init-sync\SKILL.md` 存在，必要時建立後重開 Codex |
| macOS / Linux 上「開工」沒觸發 | `startup-sync` skill 不存在或 Codex 尚未重啟 | 確認 `$HOME/.codex/skills/startup-sync/SKILL.md` 存在，必要時建立後重開 Codex |
| macOS / Linux 上「收工」沒觸發 | `shutdown-sync` skill 不存在或 Codex 尚未重啟 | 確認 `$HOME/.codex/skills/shutdown-sync/SKILL.md` 存在，必要時建立後重開 Codex |
| macOS / Linux 上「新專案初始化」沒觸發 | `project-init-sync` skill 不存在或 Codex 尚未重啟 | 確認 `$HOME/.codex/skills/project-init-sync/SKILL.md` 存在，必要時建立後重開 Codex |
| README 或 Obsidian 寫 skills 已建立，但實際沒觸發 | 文件狀態可能是舊電腦或舊 session 的紀錄 | 直接檢查 `~/.codex/skills/.../SKILL.md`，以本機檔案為準 |
| 建立 `~/.codex/skills` 時出現權限問題 | 全域 skill 在專案資料夾外，Codex 沙盒可能不能直接寫 | 先說明要建立的檔案與路徑，取得使用者授權後再寫入 |
| 不知道 skill 要寫多長 | 寫太長會佔用上下文，也容易和 AGENTS / Obsidian 雙寫 | 每個 `SKILL.md` 只保留觸發條件、流程、安全規則；專案細節留在 `AGENTS.md` 與 Obsidian |
| Obsidian 讀不到 | Obsidian MCP 尚未設定或 Codex 尚未重啟 | 確認 `~/.codex/config.toml` 有 `[mcp_servers.obsidian]`，改完要重開 Codex |
| 專案駕駛艙被建在工作資料夾 | 把 Obsidian 駕駛艙誤當成 repo 內文件 | 應改成建在 Obsidian vault，例如 `Secondbrain/<專案名稱>/專案工作流程.md`，repo 內 AGENTS.md 只記 vault 與相對路徑 |
| 只寫 `my-project/專案工作流程.md`，不知道是哪個 vault | 只記了 vault-relative path，沒記 vault 根目錄 | AGENTS.md 要同時記 `Obsidian vault` 絕對路徑與 `專案駕駛艙` vault-relative path |
| Firebase 工具看不到 | Firebase MCP 尚未安裝或 Codex 尚未重啟 | 先跑 #04.5；Windows 建議 MCP command 用 `npx.cmd`，改完要重開 Codex |
| 不想公開 repo | 不需要 GitHub Pages 或不想公開教材 | 不使用 GitHub Pages 時可建立 private repo |
| 要跨電腦 | 工作檔、設定、筆記同步來源不同 | 工作檔靠 GDrive / OneDrive，同步設定靠 chezmoi；新電腦先跑 Codex 環境懶人包 |
| 下載的 `.md` 打開是亂碼 | 下載檔或讀取工具的編碼不一致 | 回 GitHub raw 原始檔讀取，不要直接依亂碼版本修改 |
| 已有專案要套 #07 | 不是空資料夾，已有規則與歷史 | 先檢查現況，只補缺少的 `README.md`、`.gitignore`、Obsidian 紀錄、全域 skills 等，不要覆蓋既有設定 |
| `AGENTS.md` 顯示未追蹤 | 舊專案可能原本只有 `CLAUDE.md`，或 AGENTS 尚未納入 Git | 若 #07 要讓 Codex 使用 AGENTS，確認內容無敏感資訊後納入本次 commit |
| `.gitignore` 註解或中文顯示亂碼 | 舊檔案編碼或終端顯示問題 | 可保留規則本體，將短檔案整理成 UTF-8、繁中註解清楚的安全版本 |
| README 缺少 | 舊示範專案可能直接靠 HTML 與 AGENTS 運作 | 補一份簡短 README：用途、主要檔案、工作模式、安全原則即可 |
| GitHub API 更新中文檔名失敗 | PowerShell / Node 管線可能把中文檔名轉成問號 | 對 GitHub contents API 使用 URL encoded path，或用純 PowerShell Unicode 流程 |
| GitHub 文件被壓成一長行 | 讀 raw 時沒有保留換行 | 用 `$Lines = gh api ...` 後 `$Content = $Lines -join "`n"` 再更新 |

## 安全與隱私

- `.codex/`、`.claude/` 不要 commit
- `.env`、API key、token、密碼不要 commit
- Firebase 前端 config 可以公開，但 Admin 憑證不可以
- 學生資料只存座號與班級代號，不存真名
- AGENTS.md 記規則；Obsidian 記進度；不要雙寫漂移
