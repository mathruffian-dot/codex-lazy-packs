## 這個懶人包會幫你做什麼？

這是一份「工作模式啟動包」。跑完之後，你就可以用 Codex 管理一個教學應用專案：

- 建好 Google Drive / OneDrive 裡的工作資料夾
- 建好 `AGENTS.md`，讓 Codex 每次都知道專案規則
- 建好 Obsidian 專案駕駛艙，記錄進度、下一步、踩坑
- 建好 GitHub repo，必要時啟用 GitHub Pages
- 套用 Codex 的三個全域流程：
  - `project-init-sync`：新專案初始化
  - `startup-sync`：開工接續
  - `shutdown-sync`：收工同步
- 讓專案準備好，接著回到影片教學，跟著做第一個工具

> 一句話：Claude 版靠 `/開工`、`/收工` skills；Codex 版改用內建的 `開工`、`收工`、`新專案初始化` 三個全域 skill 流程。

## GitHub repo 建立流程

若要用 GitHub Pages，建議公開 repo。

PowerShell 範例：

```powershell
Set-Location "G:\我的雲端硬碟\<專案名稱>"
git init
git config windows.appendAtomically false
git add AGENTS.md README.md .gitignore
git commit -m "初始化教學工具專案"
git branch -M main
gh repo create <repo名> --public --source=. --push
```

如果不需要 Pages，可以改成私有：

```powershell
gh repo create <repo名> --private --source=. --push
```

> Google Drive 內的 git repo 建議先設 `git config windows.appendAtomically false`，可避免 `cannot update the ref` 之類的同步衝突。

## 2026-06-08 實測踩坑修正版：GitHub 上游無寫入權限時改走 fork + PR

本次實測情境：
- 本機 repo 已接上上游：`mathruffian-dot/2026database`
- 本機 commit 已完成：`77304b9 套用 Codex 專案工作模式`

```text

fatal: unable to access 'https://github.com/mathruffian-dot/2026database.git/': The requested URL returned error: 403
```

```text

Resource not accessible by integration
```

判斷方式：
```powershell

gh repo view mathruffian-dot/2026database --json viewerPermission
```

如果看到：
```json
{"viewerPermission":"READ"}
```

修正流程 A：請上游 repo 擁有者加權限
- 到 GitHub repo settings → Collaborators / Access
- 把目前帳號加成 `Write`、`Maintain` 或 `Admin`
- 再驗證：
```powershell
gh repo view mathruffian-dot/2026database --json viewerPermission
```

修正流程 B：無法取得上游寫入權限時，改走 fork + PR

## 常見問題

| 問題 | 解法 |
|---|---|
| `gh repo create` 失敗 | 先跑 `gh auth status`，未登入就 `gh auth login` |
| GDrive 內 commit 報錯 | 跑 `git config windows.appendAtomically false` 後重試 |
| 新 repo 跑 `git config windows.appendAtomically false` 出現 `fatal: not in a git directory` | 先 `git init`，再跑 `git config windows.appendAtomically false` |
| 文件寫 main 但 Git 顯示 master | 第一次 commit 後執行 `git branch -M main` |
| 「開工」沒觸發 | 確認 `C:\Users\<你>\.codex\skills\startup-sync\SKILL.md` 存在 |
| 「收工」沒觸發 | 確認 `C:\Users\<你>\.codex\skills\shutdown-sync\SKILL.md` 存在 |
| 「新專案初始化」沒觸發 | 確認 `C:\Users\<你>\.codex\skills\project-init-sync\SKILL.md` 存在 |
| Obsidian 讀不到 | 確認 `~/.codex/config.toml` 有 `[mcp_servers.obsidian]`，改完要重開 Codex |
| 專案駕駛艙被建在工作資料夾 | 這是把 Obsidian 駕駛艙誤當成 repo 內文件；應改成建在 Obsidian vault，例如 `Secondbrain/<專案名稱>/專案工作流程.md`，repo 內 AGENTS.md 只記 vault 與相對路徑 |
| 只寫 `my-project/專案工作流程.md`，不知道是哪個 vault | AGENTS.md 要同時記 `Obsidian vault` 絕對路徑與 `專案駕駛艙` vault-relative path |
| Firebase 工具看不到 | 先跑 #04.5；Windows 建議 MCP command 用 `npx.cmd`，改完要重開 Codex |
| 不想公開 repo | 不使用 GitHub Pages 時可建立 private repo |
| 要跨電腦 | 工作檔靠 GDrive / OneDrive，同步設定靠 chezmoi；新電腦先跑 Codex 環境懶人包 |
| 下載的 `.md` 打開是亂碼 | 下載檔或讀取工具的編碼不一致 | 回 GitHub raw 原始檔讀取，不要直接依亂碼版本修改 |
| 已有專案要套 #07 | 不是空資料夾，已有規則與歷史 | 先檢查現況，只補缺少的 `README.md`、`.gitignore`、Obsidian 紀錄等，不要覆蓋既有設定 |
| `AGENTS.md` 顯示未追蹤 | 舊專案可能原本只有 `CLAUDE.md`，或 AGENTS 尚未納入 Git | 若 #07 要讓 Codex 使用 AGENTS，確認內容無敏感資訊後納入本次 commit |
| `.gitignore` 註解或中文顯示亂碼 | 舊檔案編碼或終端顯示問題 | 可保留規則本體，將短檔案整理成 UTF-8、繁中註解清楚的安全版本 |
| README 缺少 | 舊示範專案可能直接靠 HTML 與 AGENTS 運作 | 補一份簡短 README：用途、主要檔案、工作模式、安全原則即可 |
| GitHub API 更新中文檔名失敗 | PowerShell / Node 管線可能把中文檔名轉成問號 | 對 GitHub contents API 使用 URL encoded path，或用純 PowerShell Unicode 流程 |
| GitHub 文件被壓成一長行 | 讀 raw 時沒有保留換行 | 用 `$Lines = gh api ...` 後 `$Content = $Lines -join "`n"` 再更新 |

## 安全與隱私

- `.codex/`、`.claude/` 不要 commit
- `.env`、API key、token、密碼不要 commit
- Firebase 前端 config 可以公開，但 Admin 憑證不可以
- 學生資料只存座號與班級代號，不存真名
- AGENTS.md 記規則；Obsidian 記進度；不要雙寫漂移
