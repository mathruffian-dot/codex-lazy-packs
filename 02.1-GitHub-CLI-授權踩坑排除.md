# Codex 懶人包 #02.1：GitHub CLI 授權踩坑排除

> 版本：v0.1（Windows / Codex Desktop 實測補充）
> 更新日期：2026-06-14

> 本篇是 #02「連接 GitHub」的實戰補充，專門處理新手最容易卡住的 `gh auth login`、一次性代碼過期、網頁登入與 CLI 授權不同步，以及如何確認 Codex 真的能讀寫 GitHub。

---

## 先講結論

GitHub 網頁右上角看到自己的頭像，只代表「瀏覽器已登入 GitHub」。

Codex 要能幫你 clone、push、開 PR，還需要完成 **GitHub CLI 授權**。成功標準不是網頁顯示登入，而是 PowerShell 裡的：

```powershell
gh auth status
```

出現類似：

```text
Logged in to github.com account your-account
Git operations protocol: https
Token scopes: 'gist', 'read:org', 'repo', 'workflow'
```

---

## 最穩的手動登入流程

### 1. 開 PowerShell

如果 Windows 搜尋會跳到網頁，改用這個方式：

1. 按 `Windows + R`
2. 輸入：

```text
powershell
```

3. 按 Enter

看到 `PS C:\Users\你的帳號>` 就是正確視窗。

---

### 2. 執行登入

在 PowerShell 輸入：

```powershell
gh auth login
```

照順序選：

```text
Where do you use GitHub?
> GitHub.com

What is your preferred protocol for Git operations on this host?
> HTTPS

Authenticate Git with your GitHub credentials?
> Yes

How would you like to authenticate GitHub CLI?
> Login with a web browser
```

---

### 3. 立刻輸入一次性代碼

PowerShell 會顯示：

```text
! First copy your one-time code: XXXX-XXXX
Open this URL to continue in your web browser: https://github.com/login/device
```

請立刻打開：

<https://github.com/login/device>

輸入 PowerShell 顯示的最新代碼。

> [!warning]
> 一次性代碼會過期。過期後不能重用，也不是帳號壞掉。重新跑 `gh auth login` 拿新代碼即可。

---

### 4. 授權 GitHub CLI

輸入代碼後，GitHub 會出現：

```text
Authorize GitHub CLI
```

確認帳號是自己的帳號後，按綠色的授權按鈕。

如果 GitHub 要你再次確認身分，可能會出現：

```text
Verify via email
```

這是正常的。去信箱完成驗證後，再回來完成授權。

---

### 5. 回 PowerShell 等成功訊息

網頁出現：

```text
Congratulations, you're all set!
Your device is now connected.
```

還不代表流程完全結束。請回到 PowerShell，等它顯示：

```text
Authentication complete.
Configured git protocol
Logged in as your-account
```

不要太早關掉 PowerShell。

---

## 驗證是否真的成功

在 PowerShell 輸入：

```powershell
gh auth status
```

成功時會看到：

```text
github.com
  Logged in to github.com account your-account
  Active account: true
  Git operations protocol: https
  Token scopes: 'gist', 'read:org', 'repo', 'workflow'
```

如果仍看到：

```text
You are not logged into any GitHub hosts.
```

表示 GitHub CLI 還沒有拿到授權。請重新跑 `gh auth login`。

---

## 常見踩坑

| 狀況 | 原因 | 解法 |
|------|------|------|
| GitHub 網頁已登入，但 `gh auth status` 說沒登入 | 只登入了瀏覽器，還沒授權 GitHub CLI | 跑 `gh auth login`，完成 `Authorize GitHub CLI` |
| 顯示 `Expired user code` | 一次性代碼過期 | 重新跑 `gh auth login`，用最新代碼 |
| Codex 幫忙跑登入時一直等 | `gh auth login` 正在等你到瀏覽器完成授權 | 手動開 PowerShell 跑，比較不會錯過代碼 |
| 不知道代碼在哪裡 | 代碼在 PowerShell 裡 `one-time code:` 後面 | 複製最新那組 `XXXX-XXXX` |
| Windows 搜尋 PowerShell 會跑去網頁 | 點到網頁搜尋或搜尋結果不穩 | 用 `Windows + R`，輸入 `powershell` |
| Git clone 到中文路徑時出現 `.git/config.lock` 或 permission 問題 | Windows / Git / 沙盒路徑對鎖檔較敏感 | 改用英文路徑，例如 `D:\codex\github-write-test` |
| 可以讀 repo，但不能推送 | token 權限不足或目標 repo 沒有寫入權限 | 檢查 `gh auth status` scope，以及 repo 權限 |

---

## 讓 Codex 做安全寫入測試

登入完成後，可以請 Codex 做一個不改內容的測試：

```text
請幫我測試 GitHub 寫入權限：
1. clone 目標 repo
2. 建立測試分支
3. 建立一個 empty commit
4. push 到 GitHub
5. 開 draft PR
不要改任何實際專案檔案
```

Codex 可用的測試指令範例：

```powershell
gh repo clone OWNER/REPO D:\codex\github-write-test
cd D:\codex\github-write-test
git switch -c codex-github-write-test
git commit --allow-empty -m "test: verify GitHub write access"
git push -u origin codex-github-write-test
gh pr create --base master --head codex-github-write-test --title "Test GitHub write access" --body "Empty commit to verify write access." --draft
```

成功代表 Codex 不只可以讀 GitHub，也可以替你推分支、開 PR。

---

## 給 Codex 的判斷規則

如果使用者說「我已經登入 GitHub 了」，不要只看瀏覽器畫面。請先檢查：

```powershell
gh auth status
```

如果要做寫入測試，優先用：

- 測試分支
- empty commit
- draft PR
- 不修改實際專案檔案

如果本地資料夾不是 Git repo，不要直接在裡面硬推。先確認目標 repo，再 clone 到乾淨資料夾。
